Claude Code Prompt Pack · Engineering

aisum.ai
給工程師的 Claude Code 指令包

這份文件包含 14 個任務，每個任務都有可直接複製貼到 Claude Code 的 prompt。任務按優先級排序：P1 急修（這週做）、P2 中級（下週做）、P3 優化（可延後）。

每個 prompt 都包含：背景脈絡 / 具體需求 / 不該做什麼 / 驗證方式。Claude Code 拿到後可立即動手實作。

如何使用這份指令包
給工程師的 Workflow
  • 挑一個任務。建議從 P1 開始，照順序做。每個任務獨立，不會互相依賴（除了任務 02 → 03 有資料庫前置關係）。
  • 打開該任務的 Prompt 區塊。每個任務有個黑色程式碼區，點右上角 COPY 按鈕複製整段。
  • 在 repo 根目錄打開 Claude Code。執行 claude 進入互動模式。
  • 貼上整段 prompt。Claude Code 會自動分析 repo 結構、找對應的檔案、提出實作計畫。
  • 確認計畫後讓它執行。Claude Code 會邊改邊報告進度。完成後執行任務區塊裡的「驗證指令」確認結果。
  • 重要：不要修改 prompt。每個 prompt 都已寫好正確的限制條件（DO / DO NOT），改了會影響執行結果。

prompt-03-deduplication.md
# Task: Deduplicate comparison reports ## Context The Feed contains multiple duplicate reports for the same property pairs (e.g., #0010 and #0011 are both グランドメゾン白金の杜ザ・タワー comparing the same properties). Users see "I just looked at this" which destroys trust. ## Requirements ### 1. URL normalization utility Create `src/lib/url-normalize.ts`: ```typescript export function normalizeUrl(url: string): string { try { const u = new URL(url); // Strip tracking params const stripParams = ['utm_source', 'utm_medium', 'utm_campaign', 'fbclid', 'gclid', 'ref']; stripParams.forEach(p => u.searchParams.delete(p)); // Lowercase host, remove trailing slash, normalize path let normalized = `${u.protocol}//${u.hostname.toLowerCase()}${u.pathname}`; normalized = normalized.replace(/\\/$/, ''); // Re-add remaining query params (sorted for stability) const params = [...u.searchParams.entries()].sort(); if (params.length > 0) { normalized += '?' + params.map(([k, v]) => `${k}=${v}`).join('&'); } return normalized; } catch { return url; } } export function makePairKey(urlA: string, urlB: string): string { const a = normalizeUrl(urlA); const b = normalizeUrl(urlB); // Sort so (A,B) and (B,A) are equivalent return [a, b].sort().join('|'); } ``` ### 2. Add unique constraint ```sql ALTER TABLE comparison_reports ADD COLUMN pair_key VARCHAR(2048), ADD COLUMN date_bucket DATE GENERATED ALWAYS AS (DATE(created_at)) STORED; -- Backfill pair_key for existing records (run as one-time migration) UPDATE comparison_reports SET pair_key = LEAST(source_url_a, source_url_b) || '|' || GREATEST(source_url_a, source_url_b); -- Add unique constraint with date bucket (allow re-comparison after 24h) CREATE UNIQUE INDEX idx_unique_pair_per_day ON comparison_reports(pair_key, date_bucket) WHERE status = 'published'; ``` ### 3. Modify compare creation endpoint Before creating a new report, check for existing: ```typescript import { makePairKey } from '@/lib/url-normalize'; async function handleCompare(req: Request) { const { urlA, urlB } = await req.json(); // ... existing whitelist validation ... const pairKey = makePairKey(urlA, urlB); const today = new Date().toISOString().split('T')[0]; // Check if same pair was compared today const existing = await db.comparisonReports.findFirst({ where: { pair_key: pairKey, date_bucket: today, status: 'published', }, }); if (existing) { // Increment view count and return existing await db.comparisonReports.update({ where: { id: existing.id }, data: { view_count: { increment: 1 } }, }); return Response.json({ reportId: existing.id, slug: existing.slug, cached: true, message: 'このペアは本日既に比較されています', }); } // ... continue with new report creation ... } ``` ### 4. Migration to clean existing duplicates ```sql -- Find duplicate groups WITH duplicate_groups AS ( SELECT pair_key, date_bucket, array_agg(id ORDER BY view_count DESC, created_at ASC) as ids FROM comparison_reports WHERE status = 'published' GROUP BY pair_key, date_bucket HAVING COUNT(*) > 1 ) -- Keep first (most viewed/oldest), archive the rest UPDATE comparison_reports SET status = 'archived', failure_reason = 'Deduplicated' WHERE id IN ( SELECT unnest(ids[2:]) FROM duplicate_groups ); -- Output count SELECT COUNT(*) FROM comparison_reports WHERE failure_reason = 'Deduplicated'; ``` ### 5. Frontend: Show "cached" message When the API returns `cached: true`, briefly show a toast: ```jsx toast({ message: 'このペアは本日既に比較されています。最新レポートを表示します。', duration: 3000, }); ``` ## DO NOT - Break existing report URLs/slugs (they must remain stable) - Delete archived reports (keep for analytics) - Allow re-comparison within the same day to bypass dedup (24h is the rule) ## Verification 1. POST same urlA + urlB twice → second call should return existing reportId with `cached: true` 2. Migration output should show duplicate count 3. Feed should no longer show duplicate reports
